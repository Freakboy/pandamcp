# PandaMCP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a local MCP server that exposes one browser automation tool layer over CDP-compatible browsers through stdio, legacy SSE, and Streamable HTTP transports.

**Architecture:** The server has three boundaries: CLI/transport startup, a unified browser service, and interchangeable browser backends. The unified browser service owns tool semantics and prefers a raw CDP backend; a Playwright backend can be selected or used later without changing MCP tools.

**Tech Stack:** TypeScript, Node.js 22, `@modelcontextprotocol/sdk@1.29.0`, Express, `playwright-core`, Vitest, raw browser-level CDP WebSocket.
